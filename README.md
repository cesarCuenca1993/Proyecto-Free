## 1.3 Ejemplos reales de aplicación de competencias

### Desarrollo del videojuego 2D

# 2. Selección y análisis del proyecto intermodular

## 2.1 Nombre del proyecto
Proyecto Intermodular DAM/DAW – Desarrollo de aplicaciones, videojuegos y administración de sistemas.

---

## 2.2 Descripción del proyecto
Este proyecto intermodular reúne diferentes áreas trabajadas durante el curso con el objetivo de desarrollar una solución tecnológica completa y orientada al entorno profesional.

El proyecto incluye:

- Desarrollo de un videojuego 2D llamado Sociedad de Almas.
- Implementación de un CRM/ERP veterinario utilizando Dolibarr.
- Creación y administración de bases de datos relacionadas con Football Manager.
- Desarrollo de procedures y triggers para automatizar procesos.
- Creación de diagramas UML y casos de uso.
- Configuración de servidores Linux y Windows Server.
- Desarrollo de una Landing Page responsive con enfoque mobile first.
- Uso de GitHub para el control de versiones.
- Elaboración de documentación técnica y presentación profesional en inglés.

---

## 2.3 Objetivos del proyecto
Los principales objetivos del proyecto son:

- Aplicar los conocimientos adquiridos durante el curso en un entorno práctico.
- Desarrollar habilidades de programación y diseño de aplicaciones.
- Aprender a utilizar herramientas profesionales de desarrollo.
- Mejorar la capacidad de resolución de problemas técnicos.
- Trabajar con bases de datos y automatización de procesos.
- Implementar sistemas de control de versiones.
- Configurar y administrar servidores.
- Crear un portfolio profesional orientado al mundo laboral.
- Fomentar la creatividad y la toma de decisiones técnicas.

## 2.4 Problema o necesidad que resuelve
El proyecto busca simular situaciones reales del entorno profesional tecnológico mediante el desarrollo de diferentes soluciones digitales.

Cada parte del proyecto cubre necesidades específicas:

## 2.7 Problemas encontrados
Durante el desarrollo del proyecto surgieron diferentes problemas técnicos y de diseño en varias áreas:

### Diseño y manual de identidad corporativa
- Dificultad para seleccionar correctamente los colores del manual de identidad corporativa.
- Problemas para combinar los colores de manera visualmente atractiva dentro de la página web.
- Dificultades en la maquetación y organización visual de los elementos de la Landing Page.
- Problemas al momento de crear los wireframes y estructurar correctamente la interfaz antes del desarrollo.

### Desarrollo en Java
- Dificultades en la gestión de eventos utilizando Java Swing.
- Problemas al implementar correctamente componentes gráficos como:
  - Panels
  - Labels
  - Buttons
- Organización de ventanas e interacción entre componentes dentro del juego 2D.

### Sistemas y servidores
- Problemas de configuración de red en las máquinas virtuales.
- En algunos casos era necesario utilizar el adaptador de red en modo “Host-Only” y en otros modo “Puente”.
- Debido a configuraciones incorrectas no existía conexión entre las máquinas y los servidores.
- Dificultades para comprobar la conectividad y comunicación entre cliente y servidor.

### Desarrollo web
- Problemas para adaptar correctamente el diseño responsive.
- Organización de los elementos visuales en diferentes tamaños de pantalla.
- Dificultades para aplicar el enfoque mobile first de manera adecuada.

---

## 2.8 Soluciones aplicadas
Para resolver los problemas encontrados durante el desarrollo del proyecto se aplicaron diferentes soluciones técnicas y de diseño:

### Diseño y manual de identidad corporativa
- Se realizaron pruebas de combinación de colores hasta encontrar una paleta visual adecuada para la identidad del proyecto.
- Se investigaron referencias de diseño web para mejorar la armonía visual de la Landing Page.
- Se reorganizó la estructura visual de la página para mejorar la distribución de los elementos.
- Se utilizaron wireframes previos para planificar mejor la interfaz antes de comenzar el desarrollo.

### Desarrollo en Java
- Se practicó el uso de eventos en Java Swing mediante pruebas básicas con teclado y botones.
- Se organizó mejor el código separando componentes gráficos y lógica principal.


### Sistemas y servidores
- Se consultó con los profesores para resolver los problemas de conexión de red entre máquinas virtuales.
- Finalmente se trabajó utilizando únicamente el adaptador de red en modo “Puente”, ya que el enunciado inicial generaba confusión.
- Después de corregir la configuración de red se logró establecer correctamente la comunicación entre cliente y servidor.
- Se realizaron pruebas de conectividad para verificar el correcto funcionamiento de los servidores.

### Desarrollo web
- Se aplicó un diseño responsive basado en el enfoque mobile first.
- Se reorganizaron los elementos visuales para adaptarlos a diferentes resoluciones.
- Se realizaron ajustes en la estructura de la Landing Page para mejorar la visualización en dispositivos móviles.
